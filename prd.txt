## 📄 최종 Electron 프로젝트 PRD (Product Requirements Document)

이 문서는 **Next.js + Node.js + Electron** 스택을 기반으로 하는 **다기능 오피스 툴킷** 프로젝트의 최종 제품 요구 사항을 정의합니다.

---

### I. 프로젝트 개요 (Project Overview)

| 항목 | 상세 내용 |
| :--- | :--- |
| **제품명 (가칭)** | 다기능 오피스 툴킷 (Multi-Tool Office Toolkit) |
| **목표 (Goal)** | 일반 사무직의 비효율적인 수동 계산 및 기록 작업을 자동화하여 **업무 질 향상** 달성. |
| **배경 (Background)** | 수동 기록 및 계산으로 인한 불편함과 정확도 문제를 해결하기 위함. |
| **가치 제안 (Value)** | 확장 가능한 데스크톱 플랫폼을 제공하여 다양한 업무 불편 요소를 하나의 앱에서 효율적으로 개선. |
| **제품 분류** | **툴 모음 앱 (Tool Collection App)** |
| **기술 스택** | **Electron, Next.js, Node.js** |
| **버전 관리** | **GitHub**를 통한 소스 코드 관리 및 협업. |

---

### II. 사용자 및 환경 (Users & Environment)

| 항목 | 상세 내용 |
| :--- | :--- |
| **타겟 사용자** | 일반 **사무직** 종사자 |
| **사용자 요구 사항** | 다양한 업무 불편 요소를 툴 형태로 하나씩 추가하며 **편의성**을 높이는 확장성 있는 플랫폼 요구. |
| **지원 플랫폼** | **Windows**, **macOS** |
| **성능 목표** | 빠른 초기 로딩 및 반응 속도 확보. 무거운 프로세스 지양. |

---

### III. 핵심 기능 명세 (Core Feature: 이름-숫자 합산 툴)

| 분류 | 상세 요구 사항 |
| :--- | :--- |
| **입력 및 처리** | 입력창에 `이름 [공백] 숫자` 형식으로 입력/붙여넣기. |
| **데이터 구분** | **줄바꿈** 기준으로 항목 분리, **공백** 기준으로 이름과 숫자 분리. |
| **처리 트리거** | 입력 내용이 변경될 때마다 **실시간**으로 합산 처리 및 출력창 업데이트. |
| **핵심 로직** | **이름이 동일**한 항목의 **숫자 값은 모두 합산** 처리. |
| **오류 처리** | `이름 [공백] 숫자` 형식을 벗어난 줄은 **자동으로 무시**하고 정상적인 줄만 처리 (별도 오류 메시지 불필요). |

---

### IV. 출력 및 활용 명세 (Output & Utility)

| 분류 | 상세 요구 사항 |
| :--- | :--- |
| **출력 형식** | 출력창에는 합산된 결과를 **표 (Table)** 형식으로 표시. |
| **정렬 및 순위** | 합산된 숫자 값을 기준으로 **내림차순 정렬**. 정렬된 순서대로 **순위 (1위, 2위...)** 표시. |
| **숫자 포맷** | 출력 표의 숫자 값은 **세 자리 콤마 (,)** 형식으로 표시 (예: 1,234,567). |
| **데이터 활용 (복사)** | 출력 결과를 클립보드에 복사 시, 순위 및 콤마 없이 **`이름 [공백] 숫자`** 포맷으로 줄바꿈 되어 복사. |
| **데이터 활용 (저장)** | 출력된 합산 및 정렬 결과를 **Excel 파일 (.xlsx)** 형식으로 저장하는 기능 제공. |
| **초기화** | 입력창의 내용을 모두 지우는 **[초기화] 버튼** 제공. |

---

### V. 사용자 경험 및 비기능적 요구 사항 (UX/Non-Functional)

#### 1. UI/UX 및 레이아웃

| 항목 | 상세 내용 |
| :--- | :--- |
| **앱 레이아웃** | 좌측에 확장 가능한 **세로 메뉴 리스트** (Discord 스타일). 중앙 메인 영역은 **입력창:출력창 = 좌우 50%:50%** 분할. |
| **확장성** | 비활성화된 메뉴 리스트의 **UI 뼈대 (Placeholder)**를 구현하여 향후 기능 추가에 대비. |
| **창 제어** | 최소화, 최대화, 앱 숨김 기능 지원. |
| **반응형** | 창 크기 조절에 따라 레이아웃이 유연하게 반응해야 함. |

#### 2. 테마 및 디자인

| 항목 | 상세 내용 |
| :--- | :--- |
| **디자인 철학** | **미니멀 디자인** (Minimal Design). |
| **UI 라이브러리** | **shadcn/ui**를 활용. |
| **모드 지원** | **다크 모드** 및 **라이트 모드** 지원. 기본값은 **시스템 설정**을 따름. |
| **색상** | 미니멀한 색상 팔레트 (라이트: 흰색 베이스, 다크: 검은색 베이스). 알록달록한 색상 금지. |

#### 3. 운영 및 지속성

| 항목 | 상세 내용 |
| :--- | :--- |
| **데이터 지속성** | 앱 종료 시, **입력창의 마지막 텍스트 내용**을 로컬에 자동 저장하고, **재실행 시 해당 데이터를 입력창에 불러와** 상태 복원. |
| **자동 업데이트** | 앱 실행 시 새로운 버전 자동 검사. 새 버전 발견 시 팝업을 띄우고, **[취소] 선택 시 앱은 즉시 종료**되어야 함. 다운로드 선택 시 자동 다운로드 후 자동 재실행. |
| **설정 페이지** | 테마 설정(라이트/다크/시스템) 옵션 및 현재 앱 버전 정보를 표시하는 기본 설정 페이지 제공. |

---

### VI. 성공 지표 및 개발 기준

| 항목 | 상세 내용 |
| :--- | :--- |
| **개발 성공 기준** | 상기 **모든 MUST 항목**의 정상 작동 및 안정적인 **Windows/macOS 빌드 및 자동 업데이트** 로직 구현 완료. |
| **비즈니스 성공 기준** | 사용자 피드백을 통한 업무 효율 및 정확도 개선 확인. |

---

이 PRD는 프로젝트의 모든 핵심 요구 사항을 정의하며, 개발팀이 작업을 시작할 수 있는 최종 가이드라인이 됩니다.

이제 이 PRD를 기반으로 개발 과정을 더욱 체계화하기 위해 **사용자 스토리**를 작성하거나, **기술 명세서** (데이터 저장 방식, 폴더 구조 등) 작성을 도와드릴 수 있습니다.
