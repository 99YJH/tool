# Git 관리 가이드

## ✅ Git에 올려야 하는 것들 (현재 포함됨)

### 필수 파일들
- ✅ `package.json` - 의존성 정보 (가장 중요!)
- ✅ `pnpm-lock.yaml` - 정확한 버전 잠금
- ✅ `tsconfig.json` - TypeScript 설정
- ✅ `next.config.js` - Next.js 설정
- ✅ `tailwind.config.ts` - Tailwind 설정
- ✅ `postcss.config.js` - PostCSS 설정
- ✅ `components.json` - shadcn/ui 설정
- ✅ `.npmrc` - pnpm 설정
- ✅ `.gitignore` - Git 제외 목록
- ✅ `electron-builder.yml` - 빌드 설정

### 소스 코드
- ✅ `electron/` - Electron 메인 프로세스 (.ts 파일)
- ✅ `src/` - Next.js 소스 코드
- ✅ `public/` - 정적 파일 (아이콘 등)
- ✅ `build/` - 빌드 설정 파일 (entitlements 등)

### 문서
- ✅ `README.md` - 프로젝트 설명서
- ✅ `prd.txt` - 제품 요구사항 문서

## ❌ Git에 올리면 안 되는 것들 (자동 제외됨)

### 의존성 및 빌드 결과물
- ❌ `node_modules/` - 너무 크고 불필요 (pnpm install로 생성)
- ❌ `dist/` - Electron 빌드 결과물 (tsc로 생성)
- ❌ `out/` - Next.js export 결과물 (next build로 생성)
- ❌ `release/` - 최종 실행 파일 (electron-builder로 생성)
- ❌ `.next/` - Next.js 빌드 캐시

### 로그 및 임시 파일
- ❌ `npm-debug.log*` - npm 디버그 로그
- ❌ `pnpm-debug.log*` - pnpm 디버그 로그
- ❌ `*.tsbuildinfo` - TypeScript 빌드 정보
- ❌ `.DS_Store` - macOS 시스템 파일

### 환경 변수 및 비밀키
- ❌ `.env*.local` - 로컬 환경 변수
- ❌ `*.pem` - 인증서 파일
- ⚠️ **절대 올리면 안 됨**: API 키, 비밀번호, 토큰

## 🔒 민감한 정보 관리

현재 프로젝트에는 민감한 정보가 없지만, 나중에 추가할 경우:

### 추가하면 안 되는 것들
```bash
# 환경 변수 파일
.env
.env.local
.env.production

# 인증서 및 키
*.pfx
*.p12
*.key
*.pem
signing-keys/

# 데이터베이스
*.db
*.sqlite

# IDE 설정 (선택사항)
.vscode/
.idea/
```

### 안전한 관리 방법
1. `.env.example` 파일 생성 (실제 값 없이 키만)
2. 실제 `.env` 파일은 로컬에만 보관
3. 팀원들은 `.env.example`을 복사해서 사용

## 📋 다른 PC에서 프로젝트 시작하기

### 1단계: 클론
```bash
git clone https://github.com/99YJH/tool.git
cd tool
```

### 2단계: 의존성 설치
```bash
# pnpm 설치
npm install -g pnpm

# 프로젝트 의존성 설치
pnpm install

# Electron 수동 설치 (필요한 경우)
npm install electron@28.3.3 --no-save
```

### 3단계: 실행
```bash
pnpm dev
```

## ⚠️ 주의사항

### pnpm-lock.yaml은 반드시 커밋!
- ✅ `pnpm-lock.yaml` 파일은 **반드시** Git에 포함
- 모든 팀원이 동일한 버전의 패키지 사용
- 빌드 재현성 보장

### node_modules는 절대 커밋 금지!
- 용량이 수백 MB ~ GB
- `pnpm install`로 자동 생성됨
- `.gitignore`에 이미 포함됨

### 빌드 결과물은 커밋 금지!
- `dist/`, `out/`, `release/` 제외
- 빌드 명령어로 생성하면 됨
- GitHub Releases에 업로드는 OK

## 🔄 자주 사용하는 Git 명령어

```bash
# 상태 확인
git status

# 변경사항 스테이징
git add .

# 커밋
git commit -m "메시지"

# 푸시
git push

# 풀
git pull

# 브랜치 확인
git branch

# 로그 확인
git log --oneline
```
